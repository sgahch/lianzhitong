## 1. 全局视觉规范 (Global Design System)

### 1.1 色彩体系 (Color Palette)

前端需定义 CSS 变量或 Tailwind 配置：

| 变量名 | 色值 (Hex) | 用途说明 |
| --- | --- | --- |
| `primary-blue` | **#1890FF** | 品牌主色，用于按钮、激活菜单背景、进度条、链接 |
| `success-green` | **#52C41A** | 状态：已完成/已通过 |
| `warning-red` | **#F5222D** | 状态：已逾期/删除按钮/高危操作 |
| `status-process` | **#1890FF** | 状态：进行中 |
| `text-primary` | **#1F1F1F** | 主要文字（标题、表格内容） |
| `text-secondary` | **#8C8C8C** | 次要文字（面包屑、说明、未选中图标） |
| `bg-body` | **#F0F2F5** | 页面整体灰色背景 |
| `bg-container` | **#FFFFFF** | 内容容器、侧边栏背景 |
| `border-color` | **#E8E8E8** | 分割线、边框 |

### 1.2 布局与栅格 (Layout & Grid)

* **整体架构**：Left Sidebar Fixed (220px) + Top Header Fixed (64px) + Scrollable Content.
* **内容区内边距**：`padding: 24px`.
* **卡片圆角**：`border-radius: 4px` (小圆角，强调商务感).

## 3. 主体内容区组件 (Main Content Components)

**文件名建议**: `AssistPartyList.vue`

### 3.1 顶部区域 (Page Header)

* **面包屑**: `首页 / 日常业务 / 协助党委`。
* **标题**: "协助党委工作" (20px, Bold)。
* **主操作**: `<Button type="primary" icon="plus">新建任务</Button>` (右上角浮动)。

### 3.2 筛选工具栏 (Filter Bar)

一个白色 Card 容器，包含两部分：

* **左侧**: `<InputSearch placeholder="搜索任务名称、负责人..." />` (宽度约 300px)。
* **右侧**:
* `<Select placeholder="全部状态" />`
* `<Button icon="export">批量导出</Button>`
* `<Button danger icon="delete">批量删除</Button>` (红色边框/文字)。



### 3.3 数据表格 (Data Table)

**核心交互**: 支持多选 (Selection Row)。

| 列名 (Header) | 数据类型 | 组件/样式表现 |
| --- | --- | --- |
| **(Checkbox)** | Selection | 多选框 |
| **任务名称** | String | 加粗显示，文本左对齐 |
| **负责人** | String | 普通文本 |
| **任务状态** | Enum | **Badge 组件** (Dot + Text)<br>

<br>- `进行中`: Blue<br>

<br>- `已完成`: Green<br>

<br>- `已逾期`: Red<br>

<br>- `待开始`: Grey |
| **截止日期** | Date | 格式：`YYYY-MM-DD` |
| **进度** | Number | **Progress Bar 组件**<br>

<br>- 宽度：150px<br>

<br>- 颜色：Blue (#1890FF)<br>

<br>- 显示右侧百分比文本 |
| **操作** | Actions | 蓝色文字链接，间距 8px<br>

<br>`Icon(Eye) 查看` `Icon(Edit) 编辑` `Icon(Delete, Red) 删除` |

### 💡 专家的建议

1. **图标库选择**：截图中的图标非常像 **Ant Design Icons**，建议直接使用该库，还原度 99%。
2. **状态映射**：前端记得写一个 `computed` 函数来处理状态码到颜色的映射（比如 `case 1: return 'processing'`），这样代码维护性更高。
3. **响应式**：虽然截图是 PC 端，但建议在代码中加入 `flex-layout` 保证在不同分辨率屏幕下表格不会崩坏。
